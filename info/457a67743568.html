Upgrading my Open Source Pi Surveillance Server with Frigate

中文摘要

这篇文章介绍了作者对其基于树莓派的开源监控服务器（使用Frigate软件）进行升级的经历。作者在2024年曾搭建了一套利用Axzez Interceptor机箱和Coral TPU进行物体检测的Frigate NVR系统，该系统实现了100%本地化运行，无需云集成，高度注重隐私。此次升级的核心目标是将原有系统进行小型化，同时保留对廉价大容量硬盘的支持和强大的AI加速能力。为此，作者采用了Exaviz公司新推出的Cruiser主板，并搭配DeskPi专为其设计的2U Mini Rack Mount原型机箱，通过使用树莓派Compute Module 5 (CM5) 完成了此次升级。 **核心主题：** 本文的核心主题是升级基于树莓派的开源Frigate智能监控服务器，旨在实现系统的小型化和更高集成度，同时保持高性能的本地AI物体检测能力和大容量存储，并确保数据隐私。 **主要观点：** 1. **本地化与隐私优先：** 作者强调其监控系统100%本地运行，无需云服务或账号，完全掌控数据，体现了对隐私的高度重视。 2. **性能与紧凑的平衡：** 升级的动力在于在缩小系统体积的同时，不牺牲大容量存储和AI加速器的性能。 3. **开源软硬件协同：** Frigate作为开源NVR软件与树莓派硬件、Coral TPU加速器结合，为构建经济高效且功能强大的监控方案提供了可能。 **关键技术点：** 1. **Exaviz Cruiser CM5载板：** 这是本次升级的“明星硬件”，专为树莓派Compute Module 5设计。 2. **CM5 PCIe带宽限制的解决方案：** Cruiser主板巧妙地解决了CM5只有一个PCIe Gen 2 x1通道的限制，通过混合PCIe和USB的方式扩展I/O。 * M.2 M-key NVMe插槽：直接连接到CM5的PCIe Gen 2 x1通道（理论上可运行Gen 3）。 * 2.5 Gbps WAN网口（RTL8156BG）：通过USB 3.0路由。 * 两个SATA连接（JMS561）：通过USB 3.0路由。 3. **DeskPi 2U Mini Rack Mount机箱：** 为Cruiser主板量身定制的原型迷你机架式外壳，有助于实现系统小型化和整洁部署。 4. **Coral TPU：** 继续沿用作为AI加速器，负责进行实时的物体检测，如识别人物、车辆等。 **结论：** 通过这次升级，作者成功地将一套功能强大、隐私优先的树莓派AI监控服务器进行了小型化，实现了在紧凑尺寸内集成高性能AI推理、大容量存储和高速网络的能力。Exaviz Cruiser主板在克服树莓派CM5硬件限制方面的创新设计，使其成为构建此类高效、本地化监控系统的理想平台。此次升级不仅达到了作者小型化的目标，也为其他追求高性能、高隐私和开源解决方案的用户提供了宝贵的实践经验和可行的硬件选择。
