Please, please, please stop using passkeys for encrypting user data

中文摘要

Simon Willison在其博客上发布了一篇紧急呼吁，强烈建议业界停止使用通行密钥（passkeys）来加密用户数据。这篇文章虽然简短，却言辞恳切，直指当前身份识别领域可能存在的严重误区，其核心观点是通行密钥并非为数据加密而设计，若用于此目的，将带来不可逆转的用户数据损失风险。 文章指出，将通行密钥用于加密用户数据的根本问题在于，用户非常容易丢失他们的通行密钥。更糟糕的是，许多用户可能并不理解一旦他们的数据通过通行密钥进行了加密，而该密钥又不幸丢失，那么这些数据将彻底无法恢复。这种缺乏用户意识和数据恢复机制的结合，将导致灾难性的后果。身份识别行业的资深专家Tim Cappalli也持相同立场，他恳求整个行业停止推动和使用通行密钥进行数据加密，强调通行密钥应专注于其作为卓越的、能够有效抵御网络钓鱼攻击的身份验证凭证的原始角色。 从技术角度看，通行密钥是基于WebAuthn标准构建的，旨在提供一种无密码、抗钓鱼的身份验证方法。它们利用非对称加密，通过设备上的密钥对来验证用户身份，私钥永远不会离开设备，从而大大提高了安全性。然而，这种设计理念与传统的数据加密密钥管理存在本质区别。数据加密通常需要考虑备份、恢复和密钥轮换等机制，以防止密钥丢失导致数据永久性失效。将通行密钥作为唯一的加密密钥，无异于将用户的数字资产置于一个单点故障的极端风险之下。一旦设备丢失、损坏或密钥管理不当，用户将面临无法挽回的数据损失，这与通行密钥提升用户体验和安全性的初衷背道而驰。 因此，这篇博客文章为技术人员和身份管理专业人士敲响了警钟。它提醒我们，尽管通行密钥在改善身份验证安全性和便捷性方面表现出色，但其功能边界需要被清晰定义和严格遵守。通行密钥的实用价值在于其强大的防钓鱼认证能力，而非作为数据加密方案。业界应避免将这种创新技术应用于不适合的场景，以免损害用户信任，并造成广泛的数据丢失问题。正确的做法是，让通行密钥专注于其作为安全、可靠的身份验证凭证，同时采用成熟且具备恢复机制的加密技术来保护用户数据，从而确保用户数据的安全与可访问性。
