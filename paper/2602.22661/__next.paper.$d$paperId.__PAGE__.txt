2:T277c,<h2>动机与问题背景</h2>
<p>扩散语言模型（Diffusion Language Models, DLMs）已成为自回归（AR）语言建模的有前景替代方案，具备迭代精炼、灵活引导和高效解码等优势。随着 LLaDA、Dream 等开源 DLM 的涌现，研究者发现这些模型共享大量相似的设计选择（训练目标、采样算法、评估流程）。然而，这些共同组件分散在各自的临时代码库中，缺乏透明实现，导致<strong>难以复现、比较和扩展</strong>。</p>
<p>核心痛点包括：</p>
<ul>
<li><strong>训练</strong>：不同 DLM 使用不同的扩散目标（Masked Diffusion、Block Diffusion 等），但训练逻辑与模型架构高度耦合，切换目标需大量重构；</li>
<li><strong>推理</strong>：各模型暴露不一致的推理 API，推理算法（如 Fast-dLLM 加速解码）无法跨模型复用；</li>
<li><strong>评估</strong>：DLM 对推理超参数<strong>极度敏感</strong>（温度、CFG、并行度、max new tokens 等），单一超参变化即可导致性能剧烈波动，但先前工作很少明确说明这一点，各论文使用不同评估工具和配置，统一对比困难。</li>
</ul>
<p>为此，作者提出 <strong>dLLM</strong>——一个开源框架，统一扩散语言建模的训练、推理和评估三大核心组件，同时保持足够灵活以支持新方法和架构。</p>
<h2>预备知识</h2>
<h3>离散扩散与 Masked Diffusion (MDLM)</h3>
<p>给定 token 序列 x = (x^1, , x^L) V^L，引入连续时间变量 t [0,1] 和特殊掩码 token m V。前向过程为吸收态掩码：每个 token x_0^i 以概率 t 被独立掩码： q(x_t^i | x_0^i) = (1-t)\,I(x_t^i = x_0^i) + t\,I(x_t^i = m)</p>
<p>模型 p_(x_0 | x_t) 预测被掩码位置的原始 token，训练目标为带时间加权的负对数似然： L_MDLM = E_t U(0,1), x_0[1t_i M_t - p_(x_0^i | x_t)]</p>
<p>其中 M_t 为时刻 t 的掩码索引集，1/t 权重平衡不同噪声水平的贡献。</p>
<h3>Block Diffusion (BD3LM)</h3>
<p>BD3LM 将自回归与扩散结合：序列被划分为 K 个不重叠块 B_1, , B_K，块间自回归生成，块内扩散并行生成。联合概率分解为 p_(x) = _k=1^K p_(x^B_k | x^<B_k)。每个块的目标： L_BD3LM = _k=1^K E_t, x_0[1t_i mask(B_k, t) - p_(x_0^i | x_t^B_k, x^<B_k)]</p>
<p>该分解允许缓存前序块的 KV，当前块内并行解码。</p>
<h2>dLLM 系统架构</h2>
<p>dLLM 围绕三个核心组件构建统一开发流水线：</p>
<h3>Trainer：统一训练接口</h3>
<p>当前开源 DLM 主要使用 MDLM 或 BD3LM 目标。dLLM 提供 <code>MDLMTrainer</code> 和 <code>BD3LMTrainer</code> 两个核心训练模块（位于 <code>dllm/core/trainers</code>），支持预训练和微调。关键设计：</p>
<ul>
<li><strong>模块化</strong>：扩散建模逻辑与模型架构解耦，新目标可以最小重构添加。如图1所示，MDLM/BD3LM 预训练、MDLM SFT、AR-to-MDLM 适配仅需局部修改（更换 Trainer、切换少量参数、包装 data collator）；</li>
<li><strong>基于 HuggingFace 生态</strong>：使用 <code>accelerate</code> 支持 FSDP/DeepSpeed 分布式训练，<code>peft</code> 支持参数高效微调，自定义 Trainer 为 HF <code>Trainer</code> 的轻量包装；</li>
<li><strong>可扩展</strong>：框架还包含 EditFlow trainer 的参考实现（支持并行插入/替换/删除操作的文本扩散）。</li>
</ul>
<h3>Sampler：统一推理接口</h3>
<p>不同 DLM 暴露不一致的推理 API。dLLM 引入轻量抽象 <code>Sampler(model).sample()</code>，将模型与推理算法解耦，支持即插即用的采样器替换。框架内置 <code>MDLMSampler</code>（标准采样）和 <code>MDLMFastdLLMSampler</code>（Fast-dLLM 加速解码），后者可作为前者的 drop-in 替换。</p>
<p><strong>终端可视化器：</strong>DLM 的 token 解码顺序并非严格从左到右，解码顺序本身是重要的分析特征。dLLM 提供终端可视化工具，展示掩码到解码 token 的演化过程。</p>
<h3>Evaluation：统一评估流水线</h3>
<p>DLM 对推理超参数极度敏感——温度设为 0.0、关闭 <code><eos></code> 抑制、调整 CFG 系数或并行度均可导致性能显著下降。dLLM 扩展 <code>lm-evaluation-harness</code> 框架，<strong>逐模型、逐任务</strong>匹配官方预处理、解码设置和后处理配置。经人工验证后，框架能忠实复现各模型报告的结果，同时支持跨模型一致对比。</p>
<h2>开放 DLM 训练配方与实验</h2>
<h3>微调大型 DLM 以获得推理能力</h3>
<p>受 d1 等工作启发，作者使用 dLLM 统一 Trainer 对 LLaDA 和 Dream 的 Base/Instruct 变体进行 MDLM SFT + LoRA 微调，数据集为 s1K。训练配置：序列长度 4096，20 epochs，学习率 10^-5，全局 batch size 32，LoRA r=128、=256，权重衰减 0.1，cosine 调度 + 10\</p>
<p>推理时前置 <code><reasoning></code> token 触发推理链。结果（表1）：</p>
<ul>
<li><strong>Instruct 模型</strong>：SFT 后一致提升——LLaDA-Instruct GSM8K 79.91 80.59，MATH500 34.80 35.40，Countdown 19.92 26.95，Sudoku 11.62 14.36；</li>
<li><strong>Base 模型</strong>：域内数学任务提升明显（LLaDA-Base GSM8K 64.67 73.62，MATH500 10.20 17.40），但域外任务（Countdown、代码）出现退化。</li>
</ul>
<h3>BERT-Chat：将 BERT 转化为扩散聊天机器人</h3>
<p>BERT 风格的双向编码器天然适合扩散生成。作者在 ModernBERT-base/large 上仅通过 MDLM SFT（<strong>无</strong>持续预训练、<strong>无</strong>架构修改）将其转化为扩散聊天机器人，训练数据为 Tulu 3 SFT + SmolTalk。配置：序列长度 1024，10 epochs，学习率 10^-4，batch size 384，8 A100。</p>
<p>评估结果（表2）：ModernBERT-large-chat 在多数基准上超越 GPT-2 和 GPT-2-medium。具体而言：BBH 25.6（vs.\ GPT-2-medium 17.8），MATH 3.6（vs.\ Qwen1.5-0.5B-Chat 2.1）。尽管与同规模 decoder-only ARLM 在 MMLU（29.6 vs.\ 39.2）和 HellaSwag（40.9 vs.\ 48.2）上仍有差距，结果证明 BERT 风格骨干是 DLM 可行但未被充分探索的起点。</p>
<h3>Tiny-A2D：将自回归 LM 转化为 DLM</h3>
<p>以 Qwen3-0.6B 为骨干，仅通过 SFT（无持续预训练）转化为 MDLM 和 BD3LM 两个变体。训练数据为 Tulu 3 SFT + SmolTalk + opc-sft-stage1\&2。MDLM 变体序列长度 1024，BD3LM 变体序列长度 512、块大小 32，64 A100 训练。</p>
<p><strong>值得注意：</strong>作者发现先前工作采用的 logits right shifting 技巧在其实验中反而导致性能下降，因此未使用。</p>
<p>评估结果（表3）：BD3LM 变体在代码生成上表现突出——HumanEval 46.3、MBPP 38.2，均超过原始 Qwen3-0.6B-Base（32.3 / 36.6），尽管仅用 SFT 训练。MDLM 变体 GSM8K 29.3，MMLU 40.0。两个 DLM 变体在知识和推理基准上仍落后于 AR 对应物（如 Qwen3-0.6B-Base MMLU 52.8），但结果证明 AR-to-diffusion 转化是构建 DLM 的<strong>实用且计算高效</strong>的路径。</p>
<h2>评估复现与 Fast-dLLM 加速</h2>
<h3>官方结果复现</h3>
<p>dLLM 的评估流水线能紧密匹配 LLaDA 和 Dream 的官方结果。LLaDA-Base：MMLU 65.9（官方 65.9），GSM8K 70.7（官方 70.3）。Dream-Base：MMLU 70.0（官方 69.5），GSM8K 77.0（官方 77.2）。差异在合理范围内，验证了评估流水线的一致性。</p>
<h3>Fast-dLLM 加速结果</h3>
<p>dLLM 复现了 Fast-dLLM 的加速效果。以 LLaDA-Instruct 512 为例：</p>
<ul>
<li><strong>+Cache</strong>（块级近似 KV 缓存）：3.0 加速，GSM8K 81.1 76.0；</li>
<li><strong>+Parallel</strong>（基于置信度的并行更新）：3.3 加速，GSM8K 77.6；</li>
<li><strong>+Both</strong>（组合）：7.8 加速，GSM8K 76.6（官方 11.0/77.2）。</li>
</ul>
<p>Dream-Base 512：Baseline GSM8K 75.7，+Both 74.5（4.4）；HumanEval 54.9，+Both 54.3（3.6）；MBPP 56.0，+Both 54.4（最高达 13.3）。结果表明加速与精度之间的权衡合理。</p>
<h2>关键发现与启示</h2>
<p><strong>DLM 评估对推理超参数极度敏感。</strong>这一事实在先前工作中很少被明确陈述。作者通过系统实验发现：关闭 <code><eos></code> 抑制、设置 CFG=0.5、温度设为 0.0、或每步并行生成 4 个 token 均可导致显著性能下降。max new tokens 的选择也直接影响计算量和测量性能。</p>
<p><strong>现有预训练模型可低成本转化为 DLM。</strong>无论是 BERT 风格编码器还是自回归 LM，仅需 SFT（无架构修改、无持续预训练）即可转化为功能性 DLM。最近的独立工作 Efficient-DLM 也验证了这种轻量转化可以匹配甚至超过从头训练的 DLM。</p>
<h2>相关工作对比</h2>
<p><strong>离散扩散文本生成</strong>：从 D3PM 的吸收态/均匀态扩展，到连续时间框架、得分/比率匹配、MDLM 简化、嵌入空间连续扩散、flow matching、块扩散，以及 AR-扩散混合架构（推测性扩散解码、规划式生成）等。</p>
<p><strong>开源 DLM</strong>：从 1.1B 参数的首次规模化，到 DiffuGPT/DiffuLLaMA（127M--7B AR 转化）、RND1（30B）、LLaDA2.0（100B 三阶段块级方案）、Dream（Qwen-2.5 适配 + 上下文自适应噪声重调度）、LLaDA（8B MDLM 从头训练），以及商业系统 Mercury。</p>
<p><strong>dLLM 的独特定位</strong>：首个统一训练/推理/评估的 DLM 框架，填补了先前各组件独立开发、接口不一致的空白。</p>
<h2>局限与未来方向</h2>
<ul>
<li>当前聚焦 MDLM 和 BD3LM 两种主要目标，更多扩散变体（如 flow matching）需持续集成；</li>
<li>尚未集成 RL 算法——待 DLM 领域出现广泛采纳的 RL 方法后再行支持；</li>
<li>评估复现依赖逐模型的人工配置验证，自动化程度有待提升；</li>
<li>小模型配方目前限于 SFT，未探索持续预训练对转化效果的进一步提升；</li>
<li>框架将持续跟进新开源模型的发布与集成。</li>
