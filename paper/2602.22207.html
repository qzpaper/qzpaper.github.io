Recovered in Translation: Efficient Pipeline for Automated Translation of Benchmarks and Datasets

Hanna Yukhymenko, Anton Alexandrov, Martin Vechev

中文摘要

## 问题动机与背景 多语言 LLM 评估的可靠性正受到翻译基准质量不一致的严重威胁。现有多语言基准（如 Global-MMLU、MuBench、Okapi）存在系统性翻译缺陷： • 语义漂移与上下文丢失：依赖旧版 MT 工具（Google Translate、DeepL）或早期模型（GPT-4），多语言能力落后于当前前沿模型； • 问题与答案分离翻译：在 Winogrande 等句子补全任务中，分离翻译导致语法性别标记泄露正确答案（如乌克兰语中动词词尾暗示答案），或产生语法不一致； • 术语错误与字面翻译：Global-MMLU/Okapi 中发现语义漂移（``lifespan'' 误译为``life cycle''）、错误术语（``水的''误用为``aquatic''）、格变位错误、罗马数字误读为代词等四类系统性错误； • 验证不足：Global-MMLU 仅 20\ 本文聚焦东欧与南欧语言（乌克兰语、保加利亚语、斯洛伐克语、罗马尼亚语、立陶宛语、爱沙尼亚语、土耳其语、希腊语），原因是：（1）这些语言具有复杂语法特征（丰富格系统、语法性别、体态动词），对上下文不一致极为敏感；（2）属于中等资源语言，前沿模型本地化仍不足；（3）已有翻译基准可供对比评估。 作者围绕三个核心研究问题展开：RQ1：现有多语言翻译基准是否可靠？RQ2：测试时计算扩展方法能在多大程度上提升翻译质量？RQ3：如何在翻译中高效融入目标语言的独特语言特征？ ## 系统架构与方法论 框架提供两种配置模式——Dataset 模式（平坦文本，流程简单）和Benchmark 模式（需保持问题-答案结构完整性），并实现四种翻译方法： ### SC：默认翻译 + 可选自检 零样本提示 LLM 翻译，可选在新会话中让模型对照原文评估并纠正结果。适合高资源语言的大规模文本翻译，成本最低。缺点：模型可能幻想不存在的错误。支持 few-shot 提示注入语言特定注意事项。 ### Best-of-N 采样（BoN） 在较高温度（0.7）下采样 N 个翻译候选，提示 LLM 按 1--10 分标准打分，选取最高分译文。无需奖励模型，保持无训练框架。成本可控但质量低于 T-RANK/USI——LLM 数值打分能力有限，且存在位置偏差（倾向高评前置候选）。 ### USI：通用自改进 \\[1pt] 图1：USI 方法工作流程。采样 N 个候选后融合为最优译文。 基于 Universal Self-Consistency 和 Fusion-of-N 的思路，但专门适配机器翻译。在高温度下采样 N 个候选翻译，然后提示评估 LLM 按照预定义标准（翻译质量、术语准确性、问题完整性、拼写、语法变格/变位正确性）将候选融合为最佳版本，仅使用各候选的最优特征。总计仅需 N+1 次模型调用，成本和时间效率高，且能有效纠正语言特定错误。局限在于融合步骤仅执行一次，评估模型倾向于关注前几个候选中发现的错误类型，可能遗漏其他问题。 ### T-RANK：翻译排名 \\[1pt] 图2：T-RANK 方法工作流程。多轮竞争排名消除位置偏差。 本文核心创新。高温度采样 N 个多样化翻译后，judge 模型通过比较排名（而非数值打分）评估候选质量。关键设计——多轮顺序排名策略：在 N 轮中，每个候选依次被放置在每个位置上恰好一次（构成循环网格），从而系统性消除位置偏差。最终将所有候选再次呈现给 judge 模型，对选出的最佳候选进行纠正与精化。 总计 2N+1 次模型调用。与 USI 的核心区别：USI 执行一次性融合，而 T-RANK 通过多轮竞争发现所有候选中的各类缺陷，改进选择和最终精化质量。实验表明非推理模型在此方法下展现出更复杂的推理能力，能发现其他方法遗漏的细微错误。 位置偏差分析（表5）：在 MMLU ENUK（n=5）上，尽管多轮轮换策略已大幅缓解偏差，输入位置 2 仍残留轻微优势（平均排名 2.06，最低），在五个最频繁排名组合中均占据第 1 名。 ### 多提示变体 USI 和 T-RANK 均支持多提示采样（p 个不同翻译提示各采样 1 个候选，含英文和目标语言提示），进一步提升多样性和质量。 ## 基准翻译实践 将 MMLU、Hellaswag、ARC-Challenge、Winogrande 翻译为 8 种语言，共计约 29,757 个样本。MMLU 使用 GPT-4o-mini 配合 T-RANK/USI 方法翻译；其余基准按语言选用 GPT-4o-mini 或 Gemini-2.0-Flash。核心设计：问题与答案在同一提示上下文中联合翻译，保持语义关系和语法一致性。针对 Winogrande，采用动词词形遮蔽（如``бу(-в/-ла/-ло/-ли)''）防止语法性别标记泄露答案。 ## 与相关工作的对比 • vs.\ MuBench：MuBench 覆盖 61 语言但采用统一自动化流程，缺乏针对特定基准和语言的灵活适配，未显式处理性别标记泄露等问题； • vs.\ Global-MMLU：依赖 Google Translate + 仅 20\ • vs.\ Okapi：使用早期 ChatGPT 翻译，未利用测试时计算扩展，不处理语言特定语法特征； • vs.\ AFSP/TEaR：AFSP 需平行语料检索，TEaR 为单候选自精化；本文通过多候选竞争/融合实现更强的错误检测能力。 WMT24++ 研究已表明前沿 LLM 在所有评估语言对上超越传统 MT 工具，进一步证实基于旧工具翻译的基准需要更新。 ## 局限性与未来方向 • 评分机制：BoN 选择使用 LLM 打分而非专用翻译质量模型（如 COMET），可能影响候选排名可靠性； • 无自适应难度估计：对所有条目统一应用相同方法，未根据文本复杂度自动选择翻译策略（短文本不需要最复杂方法）； • 闭源模型依赖：主要依赖 GPT-4o-mini 和 Gemini-2.0-Flash，开放权重模型测试有限。作者推测所提方法对零样本翻译能力较弱的开放模型收益更大，但需验证； • 语言覆盖：聚焦东南欧语言，需进一步验证在更广泛低资源语言上的泛化能力； • COMET 评估局限：训练数据领域偏差、与人类判断的潜在差异，应结合人工评估或 LLM-as-a-judge 综合评估； • 未来方向：基于翻译难度的自适应方法选择、集成专用翻译质量模型、在开放权重模型上全面评估、扩展至更多低资源语言。 ## 核心贡献总结 （1）系统性分析现有多语言基准翻译缺陷的根源；（2）提出全自动、高度可配置的翻译框架，支持四种方法灵活平衡质量与成本；（3）创新性 T-RANK 多轮竞争排名方法，通过位置轮换消除评估偏差，实现更精细的错误检测；（4）发布 8 种东南欧语言的改进基准翻译（MMLU、Hellaswag、ARC、Winogrande），证明翻译质量显著影响下游评估结果。开源框架和基准数据集，促进可复现的多语言 AI 开发。

Key Contributions

  • 提出T-RANK多轮竞争排名方法，通过循环网格策略系统性消除位置偏差，结合USI融合方法在机器翻译基准上超越基线。
  • 识别现有多语言基准（Global-MMLU、Okapi等）的四类系统性翻译错误，并提出联合翻译问题与答案的核心设计，解决语法性别标记泄露问题。
  • 构建涵盖8种语言约29,757样本的高质量多语言基准，为中等资源语言的LLM评估提供可靠数据集。
