VGGT-Det: Mining VGGT Internal Priors for Sensor-Geometry-Free Multi-View Indoor 3D Object Detection

Yang Cao, Feize Wu, Dave Zhenyu Chen, Yingji Zhong, Lanqing Hong, Dan Xu

中文摘要

## 问题动机与背景 多视角室内3D目标检测是机器人与增强现实的基础任务。现有方法严重依赖传感器提供的几何输入（精确标定的多视角相机位姿或深度图），获取成本高昂且在真实场景中常不可用，极大限制了部署能力。 本文提出更具实用性的设定：Sensor-Geometry-Free（SG-Free）多视角室内3D目标检测——完全不使用传感器提供的几何输入（无位姿、无深度）。该设定极具挑战性，因为它同时去除了多视角相机位姿和深度两大关键信息源。 \\[1pt] 图1：VGGT-Det 有效挖掘 VGGT 内部语义与几何先验，而非仅消费其预测输出，在 SG-Free 设定下显著超越竞争方法。 近期前馈式3D重建模型（DUSt3R、MASt3R、Fast3R、VGGT）表明可直接从无位姿2D图像推断3D结构。其中 Visual Geometry Grounded Transformer（VGGT）不仅能预测点云、相机位姿等3D属性，其内部中间表示还编码了丰富的语义与几何先验。然而，现有工作仅将 VGGT 作为黑盒预测器使用其输出，未能挖掘其内部学到的知识。 本文提出 VGGT-Det，首个面向 SG-Free 多视角室内3D目标检测的 Transformer 框架。核心思想是将预训练 VGGT 编码器集成到检测流水线中，并通过两个关键模块有效利用其内部先验： • Attention-Guided Query Generation（AG）：利用 VGGT 注意力图作为语义先验初始化 object query，使查询聚焦于物体区域同时保持全局空间结构； • Query-Driven Feature Aggregation（QD）：引入可学习的 See-Query，与 object query 交互以``看到''其需求，动态聚合 VGGT 多层级几何特征。 ## 系统架构 \\[1pt] 图2：VGGT-Det 整体架构。VGGT 编码器提取3D感知特征，AG 模块利用注意力语义先验生成 object query，QD 模块通过 See-Query 动态聚合多层级几何特征，解码器迭代更新 query 输出检测结果。 ### 基础骨干网络 采用编码器-解码器 Transformer 架构。给定多视角图像 \I_1, , I_V\，使用冻结的预训练 VGGT 编码器提取3D感知 token。每视角输出 T_i R^M C（M 个 token，C 维），所有视角拼接为统一表示： T_concat = [T_1; T_2; ; T_V] R^(V M) C. 基础方案借鉴 3DETR，对 VGGT 预测的点云 P_pred 执行最远点采样（FPS）选取 K 个点，编码为初始 object query Q_0。随后经 L 层 Transformer 解码器：每层先执行 query 间自注意力，再与 T_concat 做交叉注意力。最终 Q_L 送入检测头预测类别 c R^K 和边界框 b R^K 7。 ### Attention-Guided Query Generation（AG） 问题：VGGT 预测的点云是场景的稠密重建，不区分前景/背景。FPS 均匀采样导致大量 query 落在背景区域，损害检测性能。 关键观察：VGGT 编码器的注意力图天然捕获了丰富的语义信息（物体区域获得更高注意力权重），尽管 VGGT 未针对语义任务训练。 方法：设点云 P R^N 3，注意力权重 A R^N。首先归一化注意力：A_norm = (A - (A)) / ((A) - (A) + )。第一个 query 选最高注意力点：I[1] = A_norm。后续点通过融合优先级迭代选取： Priority = A_norm + _dist D_norm, 其中 _dist [0,1] 平衡语义注意力与空间分散性（最优值约 0.8），D_norm 为候选点到已选集合的归一化最小欧氏距离。每步选 I[k] = _i S Priority。该设计使 query 既聚焦语义物体区域，又保持空间多样性覆盖全场景。 ### Query-Driven Feature Aggregation（QD） 动机：VGGT 编码器逐层将2D特征提升为3D表示，每层编码不同级别的几何抽象。基础方案简单地按固定顺序（第4/11/17/23层）逐层查询，未能自适应利用多层级信息。 方法：引入共享的可学习 See-Query q_see R^C。设 VGGT 编码器 L 个中间层输出 \F_1, , F_L\，F_i R^N C。 特征聚合：See-Query 经 MLP + Softmax 生成层级注意力权重 w R^L（w_i 0，_i w_i = 1），加权聚合多层级特征： F_agg = _i=1^L w_i F_i R^N C. 交互机制：See-Query 与 K 个 object query 拼接为 Q_input R^(K+1) C，共同参与自注意力（See-Query ``看到'' query 的需求）和交叉注意力（从 F_agg 中获取几何信息）。更新后的 See-Query 传递至下一解码层，重复从公式 (12) 开始的流程。这种跨层迭代精炼使特征聚合具备动态上下文感知能力。 ## 实验设置 数据集：ScanNet（18类，家庭/办公/教室）和 ARKitScenes（17类）。评估指标为 mAP@0.25。 实现细节：VGGT 编码器冻结；256 个 object query；AdamW 优化器（lr=2.5 10^-4，weight decay =10^-4）；梯度裁剪（max norm 35）；余弦退火学习率衰减至 10^-6；8 H800 GPU 训练约 2 天。 公平对比：为实现 SG-Free 设定，所有竞争方法均使用 VGGT 预测的位姿/点云重新训练。 ## 主要实验结果 ### ScanNet 定量对比 @l >X@ 方法 & mAP@0.25 \\ ImVoxelNet & 35.2 \\ FCAF3D（VGGT点云重训） & 40.6 \\ NeRF-Det（VGGT位姿重训） & 41.2 \\ MVSDet（VGGT位姿重训） & 42.5 \\ VGGT-Det (Basic backbone) & 41.4 \\ VGGT-Det (BB + AG) & 44.2 \\ VGGT-Det (BB + AG + QD) & 46.9 (+4.4) \\ 最终模型 46.9 mAP@0.25，超越最强竞争者 MVSDet 4.4 个点，超越 FCAF3D 6.3 个点。各类别中 chair（61.9 vs 55.5）、bookshelf（36.9 vs 31.0）、table（50.3 vs 43.2）等提升显著。 ### ARKitScenes 定量对比 @l >X@ 方法 & mAP@0.25 \\ ImVoxelNet & 12.4 \\ NeRF-Det & 18.1 \\ MVSDet & 19.4 \\ VGGT-Det & 28.0 (+8.6) \\ 超越 MVSDet 8.6 个点。refrigerator（57.7 vs 25.5）、washer（56.4 vs 40.1）、toilet（55.1 vs 43.8）等类别提升巨大。部分小型/薄型物体（如 TV、stove）所有方法均表现较差，因无传感器几何下定位此类物体极其困难。 ## 消融实验 ### 各组件贡献 • AG 模块：+2.8 mAP（41.4 44.2），验证注意力语义先验有效引导 query 聚焦物体区域； • QD 模块：+2.7 mAP（44.2 46.9），See-Query 学会跨解码器层自适应聚合最相关的几何特征。 ### 特征聚合策略对比 • Vanilla-v1（仅最后一层特征）：37.5； • Vanilla-v2（4层顺序查询）：44.2； • QD（See-Query 自适应聚合）：46.9（+2.7 vs Vanilla-v2）。 多层级特征优于单层（+6.7），自适应聚合进一步优于固定顺序（+2.7）。 ### _dist 敏感性 _dist=0.5: 40.7；_dist=0.8: 46.9（最优）；_dist=0.9: 44.9。过小则过度依赖注意力忽视空间结构，过大则空间分散项主导削弱语义引导。 ### 输入帧数影响 20帧: 42.6；40帧: 45.3；60帧: 46.2；80帧: 46.9；100帧: 47.3。性能在约80帧饱和，表明80帧足以捕获场景大部分信息。所有帧数下均持续超越 MVSDet。 ### 噪声鲁棒性 对 VGGT 点云添加高斯噪声（ = range noise\_level）：FCAF3D 在 noise\_level=0.01 时从 40.6 骤降至 18.7，0.1 时归零；VGGT-Det 在 0.01 时仍保持 47.0（几乎无损），0.1 时 44.5，0.3 时仍有 34.1。这源于 AG 模块利用语义先验减少了对精确几何细节的依赖。 ### 时间与显存分析 40帧/1 H800 评估：VGGT 本身耗时 0.68s / 11.71GB。在此之上，本方法额外仅需 0.23s / 3.57GB，而 MVSDet 需 0.21s / 13.81GB。时间相当但显存减少 74\ ## 与使用传感器几何方法的差距 SG-Free 设定下本方法 46.9 mAP，使用真实传感器几何（+SG）可达 58.8。MVSDet+SG 为 56.2。本方法在两种设定下均为最优，但 SG-Free 与 SG-Based 之间仍存在约 12 点差距，这是由于传感器几何提供了强先验。作者认为 SG-Free 设定在传感器几何不可用的实际场景中具有重要价值。 ## 损失动态验证 训练过程中验证损失曲线显示：（1）加入 AG 后 GIoU 损失持续低于基础骨干，证实注意力引导有效改善定位；（2）AG+QD 在初始几个 epoch 与 AG 损失接近（See-Query 尚在学习），随后显著低于 AG，表明 See-Query 成功学会了有效聚合几何特征。 ## 讨论与局限 核心贡献：（1）首个明确定义并攻克 SG-Free 室内3D检测设定；（2）AG 模块发现并利用了 VGGT 注意力中的语义先验；（3）QD 模块通过 See-Query 实现动态多层级几何特征聚合；（4）在 ScanNet 和 ARKitScenes 上显著超越现有方法。 局限与未来方向： • VGGT 本身带来显著的运行时和显存开销（0.68s / 11.71GB），是所有 SG-Free 流水线的共性瓶颈； • VGGT 输出为归一化预测，需要数据集尺度进行反归一化，限制了完全无先验部署； • 小型/薄型/嵌入式物体（如 TV、stove）在无传感器几何下仍难以定位，所有方法均表现不佳； • 未来方向：开发更轻量的类 VGGT 模型并支持度量尺度预测，可进一步推动该方向发展。

Key Contributions

  • 提出Attention-Guided Query Generation（AG）模块，利用VGGT注意力图作为语义先验初始化object query，融合注意力权重与空间分散性使查询聚焦物体区域。
  • 设计Query-Driven Feature Aggregation（QD）模块，引入可学习See-Query动态聚合VGGT多层级几何特征，实现跨层迭代精炼的上下文感知特征聚合。
  • 建立Sensor-Geometry-Free多视角3D检测新设定，证明预训练视觉几何模型的内部先验可被有效挖掘用于下游检测，无需任何传感器几何输入。
