dLLM: Simple Diffusion Language Modeling

Zhanhui Zhou, Lingjie Chen, Hanghang Tong, Dawn Song

中文摘要

## 动机与问题背景 扩散语言模型（Diffusion Language Models, DLMs）已成为自回归（AR）语言建模的有前景替代方案，具备迭代精炼、灵活引导和高效解码等优势。随着 LLaDA、Dream 等开源 DLM 的涌现，研究者发现这些模型共享大量相似的设计选择（训练目标、采样算法、评估流程）。然而，这些共同组件分散在各自的临时代码库中，缺乏透明实现，导致难以复现、比较和扩展。 核心痛点包括： • 训练：不同 DLM 使用不同的扩散目标（Masked Diffusion、Block Diffusion 等），但训练逻辑与模型架构高度耦合，切换目标需大量重构； • 推理：各模型暴露不一致的推理 API，推理算法（如 Fast-dLLM 加速解码）无法跨模型复用； • 评估：DLM 对推理超参数极度敏感（温度、CFG、并行度、max new tokens 等），单一超参变化即可导致性能剧烈波动，但先前工作很少明确说明这一点，各论文使用不同评估工具和配置，统一对比困难。 为此，作者提出 dLLM——一个开源框架，统一扩散语言建模的训练、推理和评估三大核心组件，同时保持足够灵活以支持新方法和架构。 ## 预备知识 ### 离散扩散与 Masked Diffusion (MDLM) 给定 token 序列 x = (x^1, , x^L) V^L，引入连续时间变量 t [0,1] 和特殊掩码 token m V。前向过程为吸收态掩码：每个 token x_0^i 以概率 t 被独立掩码： q(x_t^i | x_0^i) = (1-t)\,I(x_t^i = x_0^i) + t\,I(x_t^i = m) 模型 p_(x_0 | x_t) 预测被掩码位置的原始 token，训练目标为带时间加权的负对数似然： L_MDLM = E_t U(0,1), x_0[1t_i M_t - p_(x_0^i | x_t)] 其中 M_t 为时刻 t 的掩码索引集，1/t 权重平衡不同噪声水平的贡献。 ### Block Diffusion (BD3LM) BD3LM 将自回归与扩散结合：序列被划分为 K 个不重叠块 B_1, , B_K，块间自回归生成，块内扩散并行生成。联合概率分解为 p_(x) = _k=1^K p_(x^B_k | x^<B_k)。每个块的目标： L_BD3LM = _k=1^K E_t, x_0[1t_i mask(B_k, t) - p_(x_0^i | x_t^B_k, x^<B_k)] 该分解允许缓存前序块的 KV，当前块内并行解码。 ## dLLM 系统架构 dLLM 围绕三个核心组件构建统一开发流水线： \\[1pt] 图1：统一 Trainer 接口。(a) MDLM 预训练；(b) 一行代码切换到 BD3LM；(c) MDLM SFT 的最小改动；(d) AR-to-MDLM 适配的最小改动。 ### Trainer：统一训练接口 当前开源 DLM 主要使用 MDLM 或 BD3LM 目标。dLLM 提供 MDLMTrainer 和 BD3LMTrainer 两个核心训练模块（位于 dllm/core/trainers），支持预训练和微调。关键设计： • 模块化：扩散建模逻辑与模型架构解耦，新目标可以最小重构添加。如图1所示，MDLM/BD3LM 预训练、MDLM SFT、AR-to-MDLM 适配仅需局部修改（更换 Trainer、切换少量参数、包装 data collator）； • 基于 HuggingFace 生态：使用 accelerate 支持 FSDP/DeepSpeed 分布式训练，peft 支持参数高效微调，自定义 Trainer 为 HF Trainer 的轻量包装； • 可扩展：框架还包含 EditFlow trainer 的参考实现（支持并行插入/替换/删除操作的文本扩散）。 ### Sampler：统一推理接口 不同 DLM 暴露不一致的推理 API。dLLM 引入轻量抽象 Sampler(model).sample()，将模型与推理算法解耦，支持即插即用的采样器替换。框架内置 MDLMSampler（标准采样）和 MDLMFastdLLMSampler（Fast-dLLM 加速解码），后者可作为前者的 drop-in 替换。 终端可视化器：DLM 的 token 解码顺序并非严格从左到右，解码顺序本身是重要的分析特征。dLLM 提供终端可视化工具，展示掩码到解码 token 的演化过程。 ### Evaluation：统一评估流水线 DLM 对推理超参数极度敏感——温度设为 0.0、关闭 <eos> 抑制、调整 CFG 系数或并行度均可导致性能显著下降。dLLM 扩展 lm-evaluation-harness 框架，逐模型、逐任务匹配官方预处理、解码设置和后处理配置。经人工验证后，框架能忠实复现各模型报告的结果，同时支持跨模型一致对比。 \\[1pt] 图2：Fast-dLLM 评估结果（max new tokens @ 256/512）。CACHE 使用块级近似 KV 缓存；PARALLEL 使用基于置信度的并行 token 更新；CACHE \& PARALLEL 组合两者。 ## 开放 DLM 训练配方与实验 ### 微调大型 DLM 以获得推理能力 受 d1 等工作启发，作者使用 dLLM 统一 Trainer 对 LLaDA 和 Dream 的 Base/Instruct 变体进行 MDLM SFT + LoRA 微调，数据集为 s1K。训练配置：序列长度 4096，20 epochs，学习率 10^-5，全局 batch size 32，LoRA r=128、=256，权重衰减 0.1，cosine 调度 + 10\ 推理时前置 <reasoning> token 触发推理链。结果（表1）： • Instruct 模型：SFT 后一致提升——LLaDA-Instruct GSM8K 79.91 80.59，MATH500 34.80 35.40，Countdown 19.92 26.95，Sudoku 11.62 14.36； • Base 模型：域内数学任务提升明显（LLaDA-Base GSM8K 64.67 73.62，MATH500 10.20 17.40），但域外任务（Countdown、代码）出现退化。 ### BERT-Chat：将 BERT 转化为扩散聊天机器人 BERT 风格的双向编码器天然适合扩散生成。作者在 ModernBERT-base/large 上仅通过 MDLM SFT（无持续预训练、无架构修改）将其转化为扩散聊天机器人，训练数据为 Tulu 3 SFT + SmolTalk。配置：序列长度 1024，10 epochs，学习率 10^-4，batch size 384，8 A100。 评估结果（表2）：ModernBERT-large-chat 在多数基准上超越 GPT-2 和 GPT-2-medium。具体而言：BBH 25.6（vs.\ GPT-2-medium 17.8），MATH 3.6（vs.\ Qwen1.5-0.5B-Chat 2.1）。尽管与同规模 decoder-only ARLM 在 MMLU（29.6 vs.\ 39.2）和 HellaSwag（40.9 vs.\ 48.2）上仍有差距，结果证明 BERT 风格骨干是 DLM 可行但未被充分探索的起点。 ### Tiny-A2D：将自回归 LM 转化为 DLM 以 Qwen3-0.6B 为骨干，仅通过 SFT（无持续预训练）转化为 MDLM 和 BD3LM 两个变体。训练数据为 Tulu 3 SFT + SmolTalk + opc-sft-stage1\&2。MDLM 变体序列长度 1024，BD3LM 变体序列长度 512、块大小 32，64 A100 训练。 值得注意：作者发现先前工作采用的 logits right shifting 技巧在其实验中反而导致性能下降，因此未使用。 评估结果（表3）：BD3LM 变体在代码生成上表现突出——HumanEval 46.3、MBPP 38.2，均超过原始 Qwen3-0.6B-Base（32.3 / 36.6），尽管仅用 SFT 训练。MDLM 变体 GSM8K 29.3，MMLU 40.0。两个 DLM 变体在知识和推理基准上仍落后于 AR 对应物（如 Qwen3-0.6B-Base MMLU 52.8），但结果证明 AR-to-diffusion 转化是构建 DLM 的实用且计算高效的路径。 ## 评估复现与 Fast-dLLM 加速 ### 官方结果复现 dLLM 的评估流水线能紧密匹配 LLaDA 和 Dream 的官方结果。LLaDA-Base：MMLU 65.9（官方 65.9），GSM8K 70.7（官方 70.3）。Dream-Base：MMLU 70.0（官方 69.5），GSM8K 77.0（官方 77.2）。差异在合理范围内，验证了评估流水线的一致性。 ### Fast-dLLM 加速结果 dLLM 复现了 Fast-dLLM 的加速效果。以 LLaDA-Instruct @ max new tokens 512 为例： • +Cache（块级近似 KV 缓存）：3.0 加速，GSM8K 81.1 76.0； • +Parallel（基于置信度的并行更新）：3.3 加速，GSM8K 77.6； • +Both（组合）：7.8 加速，GSM8K 76.6（官方 11.0/77.2）。 Dream-Base @ 512：Baseline GSM8K 75.7，+Both 74.5（4.4）；HumanEval 54.9，+Both 54.3（3.6）；MBPP 56.0，+Both 54.4（最高达 13.3）。结果表明加速与精度之间的权衡合理。 ## 关键发现与启示 DLM 评估对推理超参数极度敏感。这一事实在先前工作中很少被明确陈述。作者通过系统实验发现：关闭 <eos> 抑制、设置 CFG=0.5、温度设为 0.0、或每步并行生成 4 个 token 均可导致显著性能下降。max new tokens 的选择也直接影响计算量和测量性能。 现有预训练模型可低成本转化为 DLM。无论是 BERT 风格编码器还是自回归 LM，仅需 SFT（无架构修改、无持续预训练）即可转化为功能性 DLM。最近的独立工作 Efficient-DLM 也验证了这种轻量转化可以匹配甚至超过从头训练的 DLM。 ## 相关工作对比 离散扩散文本生成：从 D3PM 的吸收态/均匀态扩展，到连续时间框架、得分/比率匹配、MDLM 简化、嵌入空间连续扩散、flow matching、块扩散，以及 AR-扩散混合架构（推测性扩散解码、规划式生成）等。 开源 DLM：从 1.1B 参数的首次规模化，到 DiffuGPT/DiffuLLaMA（127M--7B AR 转化）、RND1（30B）、LLaDA2.0（100B 三阶段块级方案）、Dream（Qwen-2.5 适配 + 上下文自适应噪声重调度）、LLaDA（8B MDLM 从头训练），以及商业系统 Mercury。 dLLM 的独特定位：首个统一训练/推理/评估的 DLM 框架，填补了先前各组件独立开发、接口不一致的空白。 ## 局限与未来方向 • 当前聚焦 MDLM 和 BD3LM 两种主要目标，更多扩散变体（如 flow matching）需持续集成； • 尚未集成 RL 算法——待 DLM 领域出现广泛采纳的 RL 方法后再行支持； • 评估复现依赖逐模型的人工配置验证，自动化程度有待提升； • 小模型配方目前限于 SFT，未探索持续预训练对转化效果的进一步提升； • 框架将持续跟进新开源模型的发布与集成。

Key Contributions

  • 提供模块化统一训练接口，支持MDLM/BD3LM目标一键切换，基于HuggingFace生态实现分布式训练和参数高效微调。
  • 设计统一推理抽象Sampler，将模型与采样算法解耦，支持Fast-dLLM加速解码等推理算法即插即用替换。
  • 扩展lm-evaluation-harness实现标准化评估流水线，逐模型匹配官方超参配置，忠实复现各DLM报告结果并支持跨模型一致对比。
